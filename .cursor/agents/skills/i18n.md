# Skill: i18n (Internationalization)

## Job

Internationalization using **react-i18next**. Use when adding translations, creating i18n keys, or working with localized text in React components (`.tsx`). Triggers on translation tasks, locale management, or i18n implementation.

## Project defaults

- **Default language**: Chinese (zh-CN)
- **Framework**: react-i18next
- **Edit only**: `src/locales/default/` — **never** edit JSON files under `locales/` (generated).
- **Generate**: Run `pnpm i18n` to generate translations (or manually translate zh-CN / en-US for dev preview).

## Key naming convention

- **Flat keys** with dot notation (no nested objects):

```ts
// ✅ Correct
export default {
  'alert.cloud.action': '立即体验',
  'sync.actions.sync': '立即同步',
  'sync.status.ready': '已连接',
};

// ❌ Avoid nested objects
export default {
  alert: { cloud: { action: '...' } },
};
```

- **Pattern**: `{feature}.{context}.{action|status}`
- **Parameters**: Use `{{variableName}}` — e.g. `'alert.cloud.desc': '我们提供 {{credit}} 额度积分'`
- **Avoid key conflicts**: Prefer longer, unambiguous keys (e.g. `clientDB.solve.action` vs `clientDB.solve` so `clientDB.solve.backup.title` does not collide).

## Workflow

1. Add keys to `src/locales/default/{namespace}.ts`
2. Export new namespace in `src/locales/default/index.ts`
3. For dev preview: manually translate `locales/zh-CN/{namespace}.json` and `locales/en-US/{namespace}.json`
4. Run `pnpm i18n` to generate all languages (CI handles this automatically)

## Usage in components

```tsx
import { useTranslation } from 'react-i18next';

const { t } = useTranslation('common');
t('newFeature.title');
t('alert.cloud.desc', { credit: '1000' });

// Multiple namespaces
const { t } = useTranslation(['common', 'chat']);
t('common:save');
```

## Common namespaces

- Most used: **common** (shared UI), **chat**, **setting**
- Others: auth, changelog, components, discover, editor, electron, error, file, hotkey, knowledgeBase, memory, models, plugin, portal, providers, tool, topic
