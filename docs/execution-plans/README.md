# Execution plans

Multi-step or phase-based execution plans for features and remediation work. Use these when the orchestrator or a human needs a written plan (goals, scope, agents, tasks) before implementation.

**Naming**: `phase-N-short-name.md` or `feature-name-execution-plan.md`.

**Format**: Free-form markdown; for agent handoffs, align with the structure in `.cursor/agents/orchestrator/handoff-template.md` (Objective, ScopeIn/Out, Inputs, Constraints, ExpectedOutput, DueCondition).
