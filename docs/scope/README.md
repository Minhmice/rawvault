# Scope docs

Scope and context documents for a feature or area. Use these to give the orchestrator or specialists clear boundaries (in/out of scope, dependencies, non-goals) when starting a task.

**Naming**: `feature-name.md` or `area-scope.md`.

**Format**: Free-form markdown. Can be referenced in handoffs (e.g. "See docs/scope/upload-dispatch.md for scope").
